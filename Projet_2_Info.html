Projet 2: Site Web d'un personnage

Des experts en la matière!

Aperçu

Dans ce projet, vous allez réaliser le site Web vitrine d'un personnage fictif de votre choix sous forme de CV.

Ce projet est l'occasion pour vous de pratiquer tout ce qui a été vu dans le cours ainsi que d'utiliser votre créativité.

Je vous invite vivement à ne pas attendre d'avoir vu la matière en classe pour commencer ce projet. Vous pouvez (et devriez) faire des recherches sur internet afin de consolider vos connaissancse (N'oubliez pas de citer la source si vous prenez des morceaux de code d'internet)

Le projet est à faire en équipe de 2 ou 3 (obligatoire).

Prérequis

Avant de commencer, vous devez choisir un personnage fictif de votre choix. Celui-ci peux provenir d'un Jeu Video (Mario, Pikachu, ...), d'une série TV (Homer Simpson, Amy Santiago, ...), d'un livre (Alice, Harry Potter, ...), ou de tout autre medium de votre choix.

Vous devez faire:

  • Au minimum 3 pages différentes (et complètes).
  • Une page réalisée sous forme de CV du personnage en question. Cette page contiendra les informations suivantes:
    • Une section avec les informations personnelles du personnage (Nom, Age, Adresse, etc...).
    • Une section Education.
    • Une section Expérience.
    • Une section Varia.
    • Au moins 3 Photos du personnage
    • (Vous pouvez inventer les informations du personnage si vous n'arrivez pas à les trouver sur internet!)
  • Au moins une page contenant un formulaire.
  • Vos pages devront être écrites en Francais.
  • Vous devez publier votre site sur Webdepot.

.

Remise

Vous devez remettre:

  • Trois pages HTML. (voir détails plus haut)
  • Une ou plusieurs feuilles de styles CSS.
  • Vos fichiers Javascript.
  • Le dossier contenant le tout sous forme d'archive ZIP (Comme dans le Devoir 1), sous le format "Projet1-prenom1_nom1-prenom2_nom2-prenom3_nom3.zip"
  • Une seule personne dans l'équipe doit rendre le projet. Les autres coéquipiers doivent rendre un fichier txt contenant le prenom et le nom du coéquipier qui a remis le projet.
  • Vous devez publier votre site sur Webdepot comme vu lors de la Démonstration 9 et inclure dans un fichier "url.txt" l'url de la page d'accueil sur Webdepot.

Code

Vous avez le droit d'utiliser tout ce que vous avez vu dans le cours (et meme des librairies non vues en cours si vous le désirez)! Vous serez aussi noté sur la propreté de votre code (Autrement dit: Votre code est-il bien indenté?).

HTML & CSS

3 pages HTML.

Vous êtes encouragés à:

  • Regarder le code HTML et CSS de plusieurs sites internet afin de vous en inspirer.
  • Utiliser les balises HTML5 pour délimiter les différentes zones.
  • Utiliser Bootstrap!!! Ce devoir sera plus compliqué sans Bootstrap.

Notes

  • Votre code HTML doit être valide!!!. Vous devez utiliser le validateur W3C tel que vu en classe.

Formulaire(s)

Chaque formulaire doit être envoyé à l’adresse suivante : https://www.w3schools.com/action_page.php

Validité

Votre code HTML doit être valide.

Javascript

Vous devez placer les diverses informations de votre page CV dans un unique fichier JSON et les charger de facon dynamique dans votre page en utilisant Javscript.

Voici un exemple de fichier JSON:



{
    "Nom": "Mario",
    "Emploi": "Plombier au Royaume Champignon",
    "Image": "https://thumbs.gfycat.com/BitterLeafyAyeaye-small.gif"
}

Notes

  • Vous ne pouvez pas écrire les données directement dans votre page HTML!
  • Il faut aller chercher celles-ci dans votre fichier JSON avec Javascript et les charger dans votre page HTML en utilisant l'API de selection dans le DOM.
  • Si vous essayez d'ouvrir un fichier JSON avec Javascript, vous allez vous rendre compte que cela n'est pas possible à cause des CORS policy. La solution pour contourner ce problème est d'envoyer tous vos fichiers sur Webdepot. La procédure exact est disponible dans la rediffusion de la Démonstration 9.

Critères de correction

Outre les points mentionnés ci-dessus, le travail sera corrigé selon le bareme suivant:

  • Originalité: 10%
  • Design: 10%
  • Code HTML/CSS: 30%
  • Validité du code HTML (Validateur W3C): 10%
  • Javascipt: 20%
  • Respect des consignes de remise: 10%
  • Propreté du code HTML: 10%
